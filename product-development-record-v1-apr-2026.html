1. Project Overview

A bespoke internal web platform built for Chine Brickwork, combining a Commercial CRM for tendering and pipeline management with a Construction Management module for live site operations. Designed to eventually replace the existing Excel tender tracker and improve team visibility across projects.

Business Context

Chine Brickwork operates across a pipeline of tendered projects moving through stages from initial enquiry through pricing, submission, and award. Prior to this system, pipeline tracking was managed via a shared Excel spreadsheet and Dropbox folder structure. Key pain points were:

  • No single view of all live tenders and their current stage
  • Difficulty tracking next actions and follow-up dates
  • No history of communications with contractors/clients
  • Construction site paperwork (RFIs, CVIs, delay notices etc.) handled on paper or in disparate files
  • No multi-user access with role-based permissions

Objectives

  • Centralise tender pipeline into a live, searchable CRM dashboard
  • Auto-detect new jobs from Dropbox folder structure via daily sync
  • Provide weekly reporting on job activity for team distribution
  • Build a construction platform for site teams to submit and track forms
  • Role-based access so site managers and project managers see only their projects
  • Cloud-hosted with no on-premise infrastructure required

3. Commercial CRM Platform

File: index.html  |  URL: Production Vercel URL (root path)

Overview

The Commercial Platform is the primary daily tool for Tim and the commercial team. It displays the full tender pipeline in a Kanban-style overview, allows job records to be viewed and updated, tracks communications, and generates weekly reports.

Feature List

Overview Dashboard

  • Kanban-style column layout grouped by pipeline stage
  • Stages: Being Priced, Ready to Review, Returned Take-Off, Returned Bill, Tenders Waiting For, Future Opportunities
  • Each job card shows: CB number, contractor, project name, next action date, owner initials
  • Sort toggle — order by Next Action Date or CB Number
  • Live search box with × clear button — filters cards across all columns in real time
  • Future Opportunities merged into the overview so follow-up calls appear alongside active tenders

Job Detail Panel

  • Opens as a slide-in panel from the right when a job card is clicked
  • Editable fields: contractor, project, stage, next action, next date, owner, contact details, site address, notes
  • Status flags: OK, On Hold, Won, Lost
  • Auto-saves changes to Supabase on blur / change
  • Draft chase email button — opens the user's default email client (Outlook) pre-filled with contractor name and job details

Communications Timeline

  • Per-job log of calls, emails, and notes with date and free-text description
  • Displayed in reverse chronological order (most recent first)
  • Add entry form within the job panel
  • Stored in the comms JSONB column on the jobs table

Weekly Report

  • Dedicated tab — Weekly Report
  • Week selector dropdown showing current week + previous 4 weeks
  • Two sections: This Week's Activity (comms logged in the selected week) and Job Status Update (all active jobs by stage)
  • Job status section split by sub-headings: Being Priced, Ready to Review, New Tenders Log, Tenders Waiting For
  • Clickable rows — clicking any job in the report opens the full job detail panel
  • Fixed-width column layout for clean print alignment
  • Print / Export to PDF button

Future Opportunities

  • Separate stage bucket for prospects not yet in active tendering
  • Cards shown in Overview dashboard alongside active stages
  • Follow-up date tracked — overdue items highlighted

Navigation

  • Module tabs: Overview · Future Opportunities · Weekly Report · Construction Platform (link)
  • Construction Platform tab opens in a new browser tab

4. Construction Management Platform

File: construction.html  |  URL: Production Vercel URL /construction.html

Overview

The Construction Platform is a separate, independently authenticated module for live project management. Site teams and project managers can raise and track construction forms (RFIs, CVIs, delay notices, H&S records, etc.) against their assigned projects. Admins have full visibility and user management.

Authentication

  • Separate login screen — not shared with the Commercial Platform
  • Powered by Supabase Auth (email + password)
  • Only users with a record in cp_profiles can access the platform
  • Tim Chase and Phil Watson have admin-level access
  • All other users must be invited by an admin

Projects

Projects are defined as a static list within the application. Each project has its own isolated set of forms. The project dropdown is filtered based on the logged-in user's assigned projects (for non-admins).

Register View

  • Status options: Submitted, In Progress, Resolved, Closed
  • Email to client button — opens default email client pre-filled with form details
  • Download PDF button — generates a formatted printable PDF

Dashboard

  • Per-project summary showing count of open vs resolved forms by type
  • Red highlighting for any form type with outstanding/open items
  • Refreshes automatically when a form is submitted

Master View

  • Admin and manager view showing all projects × all form types in a single grid

  • On new user's first sign-in: pending_projects automatically converted to cp_project_assignments rows
  • User list shows all users, their role, assigned projects (active users) or pending assignment list (users not yet signed in)
  • Admin can toggle project assignments on/off for active users in real time
  • Admin can change a user's role (Project Manager, Site Manager, Contracts Manager, QS, Foreman etc.)

6. User Roles & Access Control

Commercial Platform

The Commercial Platform does not currently have user authentication. It is accessible to anyone with the URL. Access is controlled by keeping the URL private to the Chine Brickwork team.

Future consideration: Adding authentication to the Commercial Platform for staff role management is listed as a planned enhancement.

Construction Platform — Role Summary

RoleAccess LevelCurrent Users
Admin Full access: all projects, all forms, Admin panel, Master View, user management Tim Chase, Phil Watson
Manager Access to assigned projects only. Can submit and view forms. No Admin panel. Project managers and site managers (invited by admin)

Row Level Security (RLS)

Supabase RLS policies enforce access at the database level — not just the UI:

  • cp_profiles: users can read their own profile; admins can read all profiles
  • cp_project_assignments: users can read their own assignments; admins can read all
  • construction_forms: managers can only read/write forms for projects they are assigned to
  • jobs: read/write for authenticated users (Commercial Platform is open)

Invite Flow

  1. Admin opens Admin panel → Invite New User
  2. Enters: Full Name, Email, Role (job title), ticks project checkboxes
  3. Profile saved to cp_profiles with pending_projects array and status Pending sign-in
  4. Admin tells the new user to visit the Construction Platform and sign up using their email
  5. On first login, the system automatically:
    • Links their Supabase Auth UUID to the profile record
    • Creates rows in cp_project_assignments from pending_projects
    • Clears pending_projects
  6. Project dropdown is filtered to their assigned projects only

7. Dropbox Integration & Sync

Overview

A Node.js script (sync-jobs.js) runs on Tim's Windows PC each morning at 08:00 via Windows Task Scheduler. It scans defined Dropbox subfolders, reads folder names to extract job data, and upserts records into the Supabase jobs table.

Scanned Folders

Dropbox PathCRM Stage SetFolder Badge
5a / 02. Ready to ReviewReady to Review5a
5a / 04. Being PricedBeing Priced5a
5a / 05. New Tenders LogTenders Waiting For5a
5a / 07. Tenders Waiting ForTenders Waiting For5a
5b / 02. HIT LISTBeing Priced5b
5b / 03. Returned Take-OffsReturned Take Off5b
5b / 04. Returned Bills & Tender TendersReturned Bill5b
5b / 05. Addendum PricingBeing Priced5b
5b / 06. Addendum ReadyReady to Review5b

Excluded from sync: 01. Declined, 03. Archive (both in 5a)

Folder Naming Convention

Folders must follow this pattern for the sync script to parse them:

CB [number] [Contractor] - [Project Name] ([DD-MM-YY])([OWNER])
e.g.  CB 332 Willmott Dixon - Highbury Court (25-03-26)(TC)

Sync Logic

  1. Read all existing jobs from Supabase (indexed by normalised CB number)
  2. Scan each Dropbox folder in priority order
  3. Parse folder name → extract CB, contractor, project, owner, tender date
  4. Look inside folder for a Job Sheet Excel file — read contact name, mobile, email, site address
  5. If job not in database → insert new record
  6. If job already exists → update stage and folder badge; add contact if DB record has none
  7. First folder match wins — a job appearing in multiple folders only gets the highest-priority stage

Sync Schedule

  • Runs daily at 08:00 via Windows Task Scheduler on Tim's PC
  • Requires Tim's PC to be on and connected to Dropbox at 08:00
  • Can be run manually at any time: node sync-jobs.js in the project folder

10. Known Limitations & Design Decisions

ItemCurrent BehaviourRationale / Resolution
Sync requires Tim's PC sync-jobs.js runs on Tim's Windows PC via Task Scheduler — if the PC is off, no sync that day Acceptable for now. Long-term: migrate to a cloud-scheduled function (e.g. Supabase Edge Function or GitHub Actions)
Projects defined statically Project list in construction.html is hardcoded. Adding a new project requires a code change and redeploy. Projects change infrequently. Future: move to a projects table in Supabase and load dynamically
Commercial Platform — no auth Anyone with the Vercel URL can access the commercial CRM URL kept private. Auth planned for Phase 5.
Invite flow requires manual Supabase sign-up Invited users must create their own Supabase Auth account before they can log in Supabase admin invite API requires a service-role key which cannot be exposed client-side. Solution: admin sends invite email manually; user signs up at platform URL
Comms stored as JSONB Communications log is stored as an array inside the jobs record, not a separate table Simpler to implement for a single-user app. Potential future issue with concurrent edits — migrate to a comms table if concurrent use becomes common
Excel date parsing Dates in job sheet Excel files are Excel serial numbers and must be converted Handled in sync script via excelDateToString()
